模型上下文协议 (MCP) 图解指南

🌟 MCP 全景图

想象一下MCP就像是一座魔法桥梁，连接AI大脑和外部世界的各种宝藏。AI可以通过这座桥走出去，去查阅文件、搜索信息、使用工具，然后带着这些知识回来回答你的问题！

🏗️ 核心架构

Host（主机）

像Claude这样的AI应用程序，需要访问外部数据

Client（客户端）

在Host中运行的协议客户端组件，与服务器建立1:1连接

Server（服务器）

提供特定功能的轻量级程序，通过MCP标准化接口暴露能力

想象Host是一个小学生(Claude)，Client是他的笔记本，Server是图书管理员。小学生想知道恐龙的信息，就通过笔记本(Client)向图书管理员(Server)提问，图书管理员去书架(数据源)找书，然后把答案给小学生。

📦 Resources（资源）

MCP中的基本数据单元，包含结构化信息，有唯一ID和类型

Resource就像乐高积木，每个积木都有特定形状和颜色(类型)，还有独特编号(ID)。AI可以用这些积木搭建复杂的知识城堡！
资源类型举例：
  • 文本文档
  • 图像
  • 工具定义
  • 查询结果
资源特点：
  • 唯一标识符
  • 明确的类型
  • 可以嵌套
  • 可以更新

📝 Prompts（提示）

发送给AI模型的指令，包含资源和指令信息

Prompt就像是给AI写的一封信，信中既可以包含问题，也可以附上照片(图像资源)、参考文献(文档资源)和工具列表(功能资源)。

提示支持多种呈现给AI的信息，包括：文本、资源引用、历史对话等

🔧 Tools（工具）

AI可以调用的功能，类似编程语言中的函数

工具就像AI的超能力手环！当AI遇到需要计算、搜索或获取实时信息的问题时，可以按下手环上的按钮(调用工具)，立刻获得特殊能力完成任务！

🎲 Sampling（采样）

控制AI生成内容的过程参数，如温度、创造性等

采样就像烹饪时的火候控制。温度低时，AI做出的"菜肴"(回答)总是很可预测，像按照精确食谱烹饪；温度高时，AI会更有创意，像大厨即兴发挥，但可能偶尔"味道怪怪的"(不那么准确)。
低温度设置：

更确定、更保守的回答

适合需要精确信息的场景

高温度设置：

更多样、更创造性的回答

适合创意写作和头脑风暴

🌱 Roots（根源）

用于追踪AI决策和信息来源的链条机制

Root就像家谱树，帮助我们追溯AI回答中每个信息"从哪里来"。当AI说"地球是圆的"，我们可以沿着Root看到这个信息是从哪本书或哪个数据源学到的。

这对于验证AI生成内容的准确性和解释AI决策过程非常重要

🚛 Transports（传输）

MCP中数据传输的技术机制，如WebSocket、gRPC等

Transport就像不同类型的道路。有高速公路(WebSocket)适合大量快速数据传输，有乡村小路(HTTP)适合简单请求，还有特殊运输车道(gRPC)适合特定类型的数据包裹。AI和服务器之间的信息就是通过这些"道路"传递的。
WebSocket:

持久连接

适合实时双向通信

HTTP:

请求-响应模式

适合简单数据交换

gRPC:

高性能RPC框架

适合微服务架构

🔄 Function Call与MCP的关系

Function Call是MCP Tools概念的一种简化实现

如果MCP是一个完整的游乐园，有过山车、旋转木马、碰碰车和餐厅，那么Function Call就像只是其中的旋转木马。很好玩，但只是整个游乐体验的一部分！

🎓 总结

MCP是一套完整的协议，让AI能够以标准化方式与外部世界互动

MCP就像给AI打开了一扇通往真实世界的大门。以前的AI像被关在房间里的小孩，只能靠记忆中的知识回答问题；有了MCP，AI可以走出房间，查阅书籍(文件)、问专家(API)、使用工具，真正成为我们的智能助手！
